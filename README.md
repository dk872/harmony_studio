# Harmony Studio

## Опис проєкту
**Тема проєкту**: Розробка універсальної системи керування записами та профілями користувачів для салону краси.

**Метою** **нашого** **проєкту** є розробка веб-сайту салону краси, який надає зручний доступ до інформації про послуги та спеціалістів, дозволяє клієнтам швидко й просто записуватися на процедури, а також забезпечує адміністрацію салону ефективними інструментами для управління послугами, записами та новинами.

Зараз багато салонів краси мають обмежені можливості онлайн-присутності: інформація про послуги та спеціалістів подається у вигляді звичайних веб-сторінок без інтерактивності або лише в соціальних мережах. Це ускладнює процес запису на процедури, оскільки клієнтам доводиться телефонувати або писати в месенджери, що забирає час і не завжди є зручним.
Відсутність сучасної онлайн-системи призводить до таких проблем:
- складність у пошуку повної та актуальної інформації про послуги;
- відсутність можливості онлайн-запису;
- обмеженість у комунікації між клієнтом і спеціалістом;
- ускладнене керування записами та персоналом для адміністрації салону.

Проєкт нашого салону краси покликаний виправити всі ці проблеми.

Основні цілі проєкту:
- забезпечити клієнтів зручним інструментом для перегляду послуг та онлайн-запису.
- надати можливість створення особистого кабінету клієнта.
- реалізувати кабінет спеціаліста з можливістю управління власним графіком і перегляду записів.
- зробити сайт привабливим і простим у використанні для різних категорій користувачів.

## Структура проєкту
- Каталог `/front`: містить усі вихідні файли клієнтського застосунку на React та TypeScript.
  - `/src/pages`: модульне розділення UI за функціоналом.
  - `/src/components`: повторно використовувані елементи інтерфейсу.
  - `/src/context`: керування глобальним станом, зокрема AuthContext.tsx для автентифікації.
  - `/src/assets`: шрифти, зображення та стилі. 

- Каталог `/backend`: містить усі файли серверної частини на Flask.
  - `/app/models`: шар доменних моделей, що відображає сутності бази даних.
  - `/app/routes`: шар контролерів, що реалізує REST API для взаємодії з фронтендом.
  - `/migrations`: містить налаштування alembic для керування схемою бази даних MySQL.

Структура папок спрощує командну роботу, оскільки Frontend- і Backend-розробники можуть працювати у своїх окремих каталогах, дотримуючись чіткого інтерфейсу взаємодії (REST API).


## Функціональні можливості
**Авторизація та аутентифікація користувачів**. У системі зроблені сторінки реєстрації та входу, які дозволяють користувачам створювати облікові записи та входити в систему відповідно до своєї ролі - клієнта або майстра.

**Запис на послуги салону**. На сторінках послуг і профілю майстра користувач може переглядати доступні категорії процедур, обирати майстра, дату й час прийому, підтверджувати або скасовувати запис.

**Керування послугами та майстрами**. Передбачено сторінки для адміністратора, який може видаляти або редагувати послуги, а також управляти інформацією про майстрів. Майстри мають можливість самостійно формувати свій графік і редагувати доступні часові слоти.

**Керування клієнтами**. У кабінеті майстра реалізовано перегляд списку клієнтів, записаних на прийом, а також можливість скасування записів у разі неявки або змін у графіку. Адміністратор отримує доступ до загального списку користувачів.

**Перегляд новин**. Розділ `News` представлений у форматі акордеона на головній сторінці.

**Керування записами**. У кабінетах клієнта та майстра реалізовано відповідні розділи “Your appointments”, що дозволяють переглядати, додавати, редагувати або скасовувати записи.

## Виконання нефункціональних вимог
**Швидкодія**. Тестування сторінок у локальному середовищі показало, що час відгуку на основні дії (завантаження сторінки, відправлення форми) не перевищує 1 секунди.

**Завантаження сторінок**. Візуальні компоненти сайту оптимізовані, що забезпечує час завантаження менше 2 секунд.

**Інтерактивність інтерфейсу**. Реакція елементів (кнопок, акордеонів, форм) відбувається миттєво — у межах 500 мс.

**Безпека**. Аутентифікація реалізується з використанням шифрування паролів (через хешування), а обмін даними між клієнтом і сервером здійснюється без передавання чутливої інформації у відкритому вигляді.

## Інструкція для запуску
1. Клонуйте репозиторій за допомогою команди ```git clone https://github.com/dk872/harmony_studio```
2. Перейдіть у каталог бекенду ```cd .\harmony_studio\HarmonyStudio\```
3. Створіть віртуальне середовище ```python -m venv venv```
4. Активуйте віртуальне середовище ```venv\Scripts\activate``` (Windows) або ```source venv/bin/activate``` (Linux)
5. Встановіть усі необхідні бібліотеки ```Python pip install -r .\backend\requirements.txt```
6. Переконайтеся, що ваш локальний сервер MySQL запущений і налаштований (скрипт для імпорту бази даних можна знайти в кореневому каталозі репозиторію).
7. Запустіть бекенд ```python run.py```
8. Перейдіть у каталог фронтенду ```cd ..\front\```
9. Встановіть необхідні залежності ```npm install```
10. Запустіть React-додаток ```npm start```
Бекенд повинен запуститися за адресою: `http://127.0.0.1:5000`. Фронтенд автоматично відкриється у браузері за адресою: `http://localhost:3000`.
У файлі `./HarmonyStudio/backend/config.py` треба вказати правильні налаштування для підключення до бази даних, зокрема ім’я користувача, пароль, порт.  

## Висновки
Ми успішно закінчили роботу над навчальним проєктом із розробки вебзастосунку для салону краси "Harmony Studio", досягнувши поставлених цілей. Проєкт дав нам можливість засвоїти аспекти ефективної командної роботи та ключові засади Full-Stack розробки, реалізуючи сучасну, адаптивну та функціональну систему керування салоном краси.

Ключові досягнення:
- **Повний стек**: було успішно інтегровано стек React + TypeScript на фронтенді та Python + Flask + MySQL на бекенді, що забезпечило чітке розділення відповідальності та високу швидкість розробки.
- **Універсальність профілю**: реалізовано єдину універсальну сторінку профілю, яка динамічно адаптується під ролі Майстра та Клієнта, надаючи відповідний функціонал (календар, редагування біографії, перегляд записів).
- **Керування даними (CRUD)**: cтворено функціонал для Адмін-панелі, що дозволяє адміністраторам переглядати, оновлювати та видаляти (RUD) записи в усіх основних таблицях системи. Створення нових записів (C) інтегровано у відповідні клієнтські потоки (реєстрація для користувачів та сторінка бронювання для записів).
- **Синхронізація**: успішно вирішено критичні проблеми синхронізації стану між фронтендом і бекендом, зокрема, забезпечення атомарності операцій (видалення часу з бази даних під час бронювання) та надійної персистенції авторизації.

Проєкт **"Harmony Studio"** допоміг нам набути знання для створення реального комерційно орієнтованого вебзастосунку, здатного значно покращити процес взаємодії салону з клієнтами та управління внутрішніми ресурсами.
